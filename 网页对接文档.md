# OpenScanLink - 网页端对接开发文档

本文档旨在指导第三方网页系统如何通过本地接口与 **OpenScanLink** 进行通信，实现获取扫描仪设备、启动无缝连扫、获取扫描进度以及预览扫描结果等功能。

---

## 1. 基础信息

- **服务地址**: `http://127.0.0.1:17890` (默认固定端口)
- **安全校验**: 目前默认免 Token 运行，方便多个受信任的内部网页直接连接。如果需要安全限制，可以在助手的配置中后续扩展。
- **跨域与安全**: 助手已内置 CORS 跨域支持，但仅允许本地（localhost / 127.0.0.1）发起的请求。

---

## 2. API 接口说明

### 2.1 获取扫描设备列表
用于在网页端下拉框展示当前电脑连接的扫描仪。建议仅展示 TWAIN 驱动以获得最佳的高速连扫体验。

- **URL**: `/api/device/list`
- **Method**: `GET`
- **Headers**:
  - 无需额外 Header
- **响应示例**:
```json
{
  "success": true,
  "data": [
    { "id": "AU362", "name": "AU362 (TWAIN - 推荐高速)", "provider": "TWAIN", "naps2Id": "AU362" }
  ]
}
```

---

### 2.2 启动无缝高速扫描
网页端点击“开始扫描”时调用。建议在调用此接口前，先通过 2.1 接口获取当前设备列表并确保扫描仪处于就绪状态。

- **URL**: `/api/scan/start`
- **Method**: `POST`
- **Payload 参数说明** (所有参数均为可选，若不传则默认使用助手中配置的值):
```json
{
  "dpi": 200,             // 扫描分辨率，支持 200 或 300 (未传递时，默认使用软件内设置的值，通常为 200)
  "mode": "color",        // 色彩模式: "color" (彩色) | "gray" (灰度) | "bw" (黑白)
  "duplex": true,         // 是否双面扫描: true | false
  "rotate180": true,      // 是否自动翻转 180 度 (用于修正进纸方向导致的倒置): true | false
  "autoDeskew": false     // 是否自动纠偏 (摆正倾斜): true | false (强烈建议设为 false 以获得极限扫描速度)
}
// 注意：前端可以完全不传递以上参数（直接发送 `{}`），此时所有参数都将默认使用桌面助手中的设置项。
```
- **响应示例**:
```json
{
  "success": true,
  "taskId": "task_1720944000000"
}
```

---

### 2.3 查询扫描进度/状态
启动扫描后，应以 1~2 秒的间隔轮询此接口，直到状态变为 `finished` 或 `error`。
注意：当状态为 `scanning` 时，`files` 数组会实时增量返回已经处理完成的图片路径，前端可以据此实现“扫一张出一张”的流式渲染。

- **URL**: `/api/scan/status/:taskId`
- **Method**: `GET`
- **响应示例**:
```json
{
  "success": true,
  "status": "scanning",
  "progress": 50,
  "message": "已处理并保存第 2 页...",
  "files": [
    "C:\\Users\\Public\\EduScan\\data\\images\\scan_20260715_123456_001.jpg",
    "C:\\Users\\Public\\EduScan\\data\\images\\scan_20260715_123457_002.jpg"
  ]
}
```

---

### 2.4 获取扫描结果（图片预览与下载）
由于浏览器严格的沙盒机制，网页无法直接通过 `file:///C:/Users/...` 渲染本地图片。必须通过助手的代理接口来加载图片。此接口不仅支持预览，也支持作为文件流供前端下载或直接推送到云端。

- **URL**: `/api/file/view`
- **Method**: `GET`
- **URL 参数**:
  - `path`: 绝对文件路径 (必须进行 URL 编码，例如使用 `encodeURIComponent()`)
- **使用示例** (直接赋值给 `<img>` 的 `src` 属性进行预览):
```html
<!-- Vue 示例 -->
<img :src="`http://127.0.0.1:17890/api/file/view?path=${encodeURIComponent(filePath)}`" />
```
**进阶用法 (将本地图片转为 File/Blob 对象上传服务器)**:
```javascript
const response = await fetch(`http://127.0.0.1:17890/api/file/view?path=${encodeURIComponent(filePath)}`);
const blob = await response.blob();
const file = new File([blob], 'scan_image.jpg', { type: 'image/jpeg' });
// 此时可将 file 对象加入 FormData 并通过 axios 提交到您的业务后端
```

---

### 2.5 终止/取消扫描 (新增)

- **URL**: `/api/scan/cancel/:taskId`
- **Method**: `POST`
- **描述**: 在扫描进行中时，随时强制终止扫描仪进纸，并杀掉底层扫描进程，释放设备占用。
- **Headers**:
  - 无需额外 Header
- **响应示例**:
```json
{
  "success": true,
  "message": "扫描任务已终止"
}
```

## 3. 前端调用完整示例 (JavaScript/Axios)

```javascript
const API_BASE = 'http://127.0.0.1:17890';

const scannerApi = axios.create({
  baseURL: API_BASE
});

async function startQuickScan() {
  try {
    // 1. 启动扫描 (参数均可选，不传则默认使用软件设置)
    const startRes = await scannerApi.post('/api/scan/start', {});

    const taskId = startRes.data.taskId;
    console.log('扫描任务已下发，任务ID:', taskId);

    // 2. 轮询状态
    const timer = setInterval(async () => {
      const statusRes = await scannerApi.get(`/api/scan/status/${taskId}`);
      const data = statusRes.data;
      
      console.log(`进度: ${data.progress}% - ${data.message}`);
      
      // 实时渲染已出的图片（扫一张出一张）
      if (data.files && data.files.length > 0) {
        console.log('当前已处理完毕的图片数:', data.files.length);
        // 此处可将 data.files 映射到页面的 img 标签进行实时预览
      }

      if (data.status === 'finished') {
        clearInterval(timer);
        console.log('扫描全部完成！最终文件路径数组:', data.files);
        
        // 3. 渲染预览图
        data.files.forEach(filePath => {
          const previewUrl = `${API_BASE}/api/file/view?path=${encodeURIComponent(filePath)}`;
          console.log('图片预览地址:', previewUrl);
          // 此时可以将 previewUrl 赋值给 img 标签展示，或将其上传至您的云服务器
        });
      } else if (data.status === 'error') {
        clearInterval(timer);
        console.error('扫描发生错误:', data.message);
      }
    }, 1500);

  } catch (err) {
    console.error('扫描服务请求失败，请检查 OpenScanLink 客户端是否已启动', err);
  }
}
```

## 4. 常见问题排查 (FAQ)

#### Q1: 为什么使用 `http://localhost:17890` 请求会很慢或者超时？
**A**: Windows 系统的底层网络栈在解析 `localhost` 时，经常会先尝试解析到 IPv6 地址 `::1`，等待失败后才会回退到 IPv4 `127.0.0.1`，这会导致每次请求凭空多出 1-2 秒的延迟。
**解决方案**: 无论在什么情况下，请**强制使用** `http://127.0.0.1:17890` 而不是 `http://localhost:17890`。目前客户端底层也已经默认开启了全网卡双栈监听，配合使用 `127.0.0.1` 即可实现毫秒级响应。

1. **调用报错 `ERR_CONNECTION_REFUSED`**
   - **原因**: 桌面端“OpenScanLink”未启动，或被杀毒软件拦截了 17890 端口。
   - **解决**: 请确保桌面客户端已打开，且能在系统右下角托盘看到图标。

2. **图片加载失败 (403 Forbidden 或 黑屏)**
   - **原因**: 提供的路径不存在，或试图访问允许范围以外的文件。
   - **解决**: 检查 `<img src="...">` 的拼接逻辑和路径参数。

3. **HTTPS 网页调用 HTTP 接口被拦截 (Mixed Content)**
   - **原因**: 现代浏览器禁止 HTTPS 网页直接发 HTTP 请求。
   - **解决**: 
     - 方案A：在客户机的浏览器设置中，将您网页的域名添加到“允许不安全内容”的白名单中。
     - 方案B：通过您网页自带的 HTTPS Node.js/Nginx 服务，做一层本地的反向代理。

# OpenScanLink - 版本更新对接文档 (自定义 JSON 方案)

本文档详细说明了桌面端应用（Electron）基于自定义 JSON 文件的自动更新机制。

## 1. 更新机制原理
本项目**废弃了**受限的 `electron-updater`（无需生成和校验 `latest.yml`），改用高度自定义的纯网络拉取与文件覆盖策略。工作原理如下：

1. **配置拉取**：客户端点击“检查更新”时，使用 `axios` 直接拉取指定的 JSON 配置文件（如 `version.json`）。
2. **版本对比**：客户端将 JSON 中的 `windowVersion` / `macVersion` 与当前运行环境的真实版本（`1.0.0`）进行**语义化版本对比**（例如 `1.1.0` > `1.0.0`）。
3. **流式下载**：若发现新版本，前端触发下载状态，主进程通过 `axios` Stream 流将 `windowUrl` 指向的安装包下载至 Windows 系统的临时目录（`os.tmpdir()`），并实时通过 IPC 广播进度给前端进度条。
4. **覆盖安装**：下载完毕后，通过 Node.js 原生的 `child_process.spawn` 独立执行该 `.exe` 文件，并自动执行 `app.quit()` 退出当前进程，从而让安装包顺利接管完成覆盖安装。

---

## 2. 运维与版本发布流程

由于不再依赖构建工具强制的产物，运维发布流程变得极其简单且自由：

1. **打包应用**：在项目中执行 `npm run build:win` 打包出最新的安装包（如 `OpenScanLink-Setup-1.1.0.exe`）。
2. **上传安装包**：将该 `.exe` 文件上传至您的任何服务器或 CDN 中。
3. **修改 JSON 配置**：更新您服务器上的 `version.json` 文件内容。例如：
   ```json
   {
     "windowUrl": "https://your-server.com/downloads/OpenScanLink_Setup_v1.1.0.exe",
     "windowVersion": "1.1.0",
     "macUrl": "https://your-server.com/downloads/OpenScanLink_Installer_Final.dmg",
     "macVersion": "1.1.0"
   }
   ```
4. **发布生效**：客户端下一次点击“检查更新”时，即会读取到 `1.1.0` 版本并触发自动下载和安装。
